# 거래(Transaction) 기능요구서

> 담당: **D - 이동준** · 도메인 패키지: `domain/transaction` · 테이블: `transactions`
> 연동: 상품 상태 변경(B - 백성민), 거래 완료 후 리뷰(E - 장인호)

---

## 1. 개요

구매자가 판매 중인 상품에 대해 거래를 신청하고, **직거래 / 택배** 방식으로 거래를 진행·완료·취소하는 기능을 정의한다.
PG(결제대행) 연동은 범위에서 제외하며, **결제 상태와 결제 정보만** 관리한다.

---

## 2. 상태 정의

### 2.1 상품 상태 (`Product.ProductStatus`)

| 값 | 의미 | 비고 |
|----|------|------|
| `SELLING` | 판매중(가능) | 거래 신청 가능 |
| `RESERVED` | 예약중 | 거래 신청됨, 다른 거래 신청 불가 |
| `SOLD` | 판매완료 | 거래 완료, 변경 불가 |
| `DRAFT` | 임시저장 | 노출 안 됨 |

### 2.2 거래 방식 (`Transaction.TransactionType`)

| 값 | 의미 |
|----|------|
| `DIRECT` | 직거래 |
| `DELIVERY` | 택배 거래 |

### 2.3 거래 상태 (`Transaction.TransactionStatus`)

| 값 | 의미 | 단계 |
|----|------|------|
| `PENDING` | 거래 요청(신청) | 구매자가 신청한 직후 |
| `AGREED` | 거래 확정 | 판매자 수락 |
| `IN_TRANSIT` | 배송 중 | 택배 거래 전용 |
| `COMPLETED` | 거래 완료 | 직거래/택배 모두 |
| `CANCELLED` | 거래 취소 | 양측 취소 |

### 2.4 택배 배송 상태 (`Transaction.deliveryStatus`)

| 값 | 의미 |
|----|------|
| `READY` | 배송준비 |
| `IN_TRANSIT` | 배송중 |
| `DELIVERED` | 배송완료 |

## 3. 기능 요구사항

| ID | 기능 | 요구사항 | 행위자 | 전제조건 |
|----|------|----------|--------|----------|
| FR-01 | 거래 생성 | 구매자는 `SELLING` 상태의 상품을 선택하여 거래를 신청할 수 있다. | 구매자 | 상품 상태 = `SELLING` |
| FR-02 | 거래 방식 선택 | 거래 신청 시 `DIRECT`(직거래) 또는 `DELIVERY`(택배) 중 하나를 선택해야 한다. | 구매자 | FR-01 |
| FR-03 | 결제 수단/상태 관리 | PG 연동 없이 결제 상태와 결제 정보(금액 등)만 저장·관리한다. | 시스템 | - |
| FR-04 | 수수료 계산 | 상품 가격 기준으로 거래 수수료를 자동 계산하고, **상품 가격 / 수수료 / 최종 결제 금액**을 사용자에게 표시한다. | 시스템 | FR-01 |
| FR-05 | 중복 거래 방지 | 상품 상태가 `RESERVED` 또는 `SOLD`인 경우 신규 거래 신청을 막는다. | 시스템 | - |
| FR-06 | 직거래 장소 지정 | 직거래 선택 구매자는 지도/주소 검색으로 희망 거래 장소(위도·경도·주소·시간)를 입력할 수 있다. | 구매자 | `type = DIRECT` |
| FR-07 | 직거래 완료 처리 | 직거래 완료 시 거래 상태를 `COMPLETED`로 변경할 수 있다. | 구매자/판매자 | `type = DIRECT`, `status = AGREED` |
| FR-08 | 택배 정보 입력 | 택배 거래의 판매자는 택배사 운송장 번호를 입력할 수 있다. | 판매자 | `type = DELIVERY` |
| FR-09 | 배송 상태 관리 | 택배 거래는 배송준비 → 배송중 → 배송완료 상태를 관리할 수 있다. | 판매자 | `type = DELIVERY` |
| FR-10 | 거래 취소 | 거래 진행 중 구매자 또는 판매자는 거래를 취소(`CANCELLED`)할 수 있다. | 구매자/판매자 | `status ∈ {PENDING, AGREED, IN_TRANSIT}` |
| FR-11 | 거래 상태 조회 | 구매자/판매자는 본인의 거래 내역과 현재 거래 상태를 조회할 수 있다. | 구매자/판매자 | 본인 거래만 |

## 4. 수수료 계산 정책 (FR-04)

```
수수료      = 상품 가격 × 수수료율
최종 결제 금액 = 상품 가격 + 수수료   (구매자 부담 기준)
```

- 수수료율 기본값: **3.5%** *(팀 협의로 확정 필요 — TBD)*
- 화면 표시 항목: 상품 가격, 수수료, 최종 결제 금액 3가지를 분리 표시
- 금액 타입은 `BigDecimal` 사용(부동소수 오차 방지), 원 단위 절사/반올림 규칙은 협의 후 확정

> 부담 주체(구매자 부담 vs 판매자 차감)는 정책 결정 사항. 본 문서는 **구매자 부담**을 기본 가정한다.

## 5. 거래 상태 흐름 (State Flow)

```
[상품 SELLING]
      │  FR-01 거래 신청
      ▼
   PENDING ──(판매자 수락)──▶ AGREED
      │                         │
      │                         ├─ 직거래: ──(FR-07 완료)──▶ COMPLETED
      │                         │
      │                         └─ 택배: ──(FR-08 운송장)──▶ IN_TRANSIT ──(배송완료)──▶ COMPLETED
      │
      └────────────── FR-10 취소 ──────────────▶ CANCELLED
                     (PENDING/AGREED/IN_TRANSIT 단계에서 가능)
```

### 상품 상태 연동 (B ↔ D)

| 거래 이벤트 | 상품 상태 변경 |
|-------------|----------------|
| 거래 신청(`PENDING`/`AGREED`) | `SELLING` → `RESERVED` |
| 거래 완료(`COMPLETED`) | `RESERVED` → `SOLD` |
| 거래 취소(`CANCELLED`) | `RESERVED` → `SELLING` (상품 복구) |

---

## 6. 중복 거래 방지 규칙 (FR-05)

- 전제: 상품 상태는 `SELLING`(가능) / `RESERVED`(예약) / `SOLD`(완료).
- 상품 상태가 `RESERVED` 또는 `SOLD`이면 **신규 거래 신청 불가** (FR-01에서 차단).
- 동일 상품에 대해 **진행 중(PENDING/AGREED/IN_TRANSIT) 거래는 1건만** 허용.

### 미결정: 예약/완료가 취소되면?

| 케이스 | 권장 처리안 |
|--------|-------------|
| `RESERVED` 상품의 거래가 취소됨 | 상품을 `SELLING`으로 **복구** → 다시 거래 신청 가능 |
| `SOLD`(완료) 후 되돌리기 | 원칙적으로 **불가**. 필요 시 "거래 취소/반품" 별도 정책으로만 처리 |

> **결정 필요(TBD)**: `SOLD` → `SELLING` 복구를 허용할지, 관리자 권한으로만 가능하게 할지.

---

## 7. 거래 취소 / 반품 정책 (FR-10)

- 취소 가능 단계: `PENDING`, `AGREED`, `IN_TRANSIT`
- 취소 불가 단계: `COMPLETED` (완료 후에는 일반 취소 불가)

### 미결정: 반품(완료 후 취소) 처리

- 완료(`COMPLETED`) 거래의 되돌림은 일반 "취소"와 구분되는 **반품** 흐름이 필요.
- **결정 필요(TBD)** 사항:
  - 반품 사유 입력 필요 여부
  - 반품 시 상품 상태 복구(`SOLD` → `SELLING`) 허용 여부
  - 환불(결제 정보) 상태 표기 방식
  - 반품 승인 주체(판매자 / 관리자)

> 1차 범위에서는 **완료 전 취소만 지원**하고, 반품은 후속 과제로 분리할 것을 권장한다.

## 8. 데이터 항목 (`transactions` 테이블)

| 필드 | 타입 | 설명 |
|------|------|------|
| `id` | Long | PK |
| `productId` | Long | 거래 상품 |
| `buyerId` | Long | 구매자 |
| `sellerId` | Long | 판매자 |
| `type` | enum | `DIRECT` / `DELIVERY` |
| `status` | enum | 거래 상태 |
| `amount` | BigDecimal | 결제 금액(최종) |
| `meetingLocation` | String | 직거래 주소 |
| `meetingLatitude` | Double | 직거래 위도 |
| `meetingLongitude` | Double | 직거래 경도 |
| `meetingTime` | LocalDateTime | 직거래 약속 시간 |
| `trackingNumber` | String | 택배 운송장 번호 |
| `deliveryStatus` | String | 배송 상태 |
| `createdAt` / `updatedAt` | LocalDateTime | 생성/수정 시각 |
